# Jellyfin.Xtream v0.9 smoke test

Run this checklist on a disposable Jellyfin 10.11 server before publishing v0.9. Use a test Xtream account that has Live TV, one catch-up channel, one movie, and one series. Back up the Jellyfin data directory and any existing export roots first.

Record the Jellyfin version, plugin commit, client used, provider type, reverse-proxy setup, and pass/fail evidence for each section. Remove usernames, passwords, proxy grants, and provider URLs from screenshots and logs.

## 1. Install and configure

- Build the final commit in Release mode and install `Jellyfin.Xtream.dll` in a clean plugin directory.
- Restart Jellyfin and confirm the plugin loads without warnings or dependency errors.
- Save valid provider credentials and use the provider connection test.
- If Jellyfin is behind a reverse proxy, set `Public Jellyfin URL` to the externally reachable HTTPS origin. Confirm invalid URLs are rejected.

## 2. Live TV naming and playback

- Configure a scoped `[LiveChannel]` cleanup rule that visibly changes a channel name; select that channel and save.
- Refresh the guide and confirm Jellyfin shows the cleaned display name. Confirm the provider ID and playback still refer to the original channel.
- Add a manual TV Override and confirm it wins over the regex-cleaned name.
- Confirm Live TV creates no STRM files.
- Play the channel for several minutes, seek/reconnect where the client permits, then stop it. Confirm Jellyfin invoked the plugin stream-opening path and released the session without buffer or disposal errors.
- Attempt an unselected/crafted channel ID through the plugin API and confirm it is rejected without a targeted provider stream request.

## 3. Guide and catch-up

- Refresh guide data containing plain and Base64 titles; verify both display correctly.
- Play a selected archive-enabled catch-up item and seek within it.
- Confirm a non-archive, unselected, or crafted catch-up request is rejected.

## 4. VOD and series channels

- Enable VOD and Series channels, including one item selected individually and one selected through its category.
- Confirm cleaned movie, series, season, and episode display names are correct and equal cleaned titles do not hide distinct provider IDs.
- Play the selected movie and episode. Confirm unselected/crafted movie, series, season, and episode IDs are rejected.
- Temporarily make provider category loading fail and confirm visibility/export settings can still be saved without clearing the existing selection.

## 5. STRM export and scanning

- Use new empty movie and series export roots. Export and confirm stable ID-bearing paths are created for selected movies and episodes only.
- Inspect several STRM files: they must contain Jellyfin resolver URLs and no Xtream base URL, username, or password.
- Add both roots as Jellyfin libraries, scan them, and play one movie and episode through the STRM libraries from an external client. This must use the configured public Jellyfin URL when set.
- Rename/clean a title and export again. Confirm the manifest reconciles the plugin-owned path without deleting an unrelated manual file placed in the root.
- Rotate the playback key and confirm an old active playback URL fails while a newly opened stream works.
- Rotate the persistent STRM key and confirm an old exported STRM grant fails; re-export and confirm playback works again.

## 6. Legacy v0.8 migration

- On a disposable root, place representative v0.8.4-v0.8.7 ID-less and v0.8.8-v0.8.12 ID-bearing movie/episode STRMs, including an old/rotated credential URL and an unrelated manual STRM.
- Preview migration from the dashboard. Confirm every expected legacy path is listed, the manual file is absent, and no file moves during preview.
- Quarantine the reviewed preview. Confirm only the reviewed unchanged files move under `.jellyfin-xtream-legacy-quarantine`, receive `.quarantined` suffixes, and appear in the completed report.
- Confirm an edited file or changed export root invalidates/skips the reviewed operation.
- Verify the new v0.9 export, then securely remove the quarantine batch because its contents retain provider credentials.

## 7. Network and log safety

- Exercise provider endpoints that return a same-origin redirect and confirm playback succeeds.
- Confirm cross-origin, metadata/link-local, and public-host-to-private redirect attempts are rejected. Confirm an explicitly configured local/LAN provider still works if that deployment is supported.
- Search Jellyfin logs for the provider username, password, credential-bearing Xtream request paths, and proxy-grant query values. None should be logged by the plugin or .NET HTTP diagnostics.
- Check resolver and proxy responses for `Cache-Control: no-store, private`, `Pragma: no-cache`, and an expired `Expires` value.

## 8. Final gate

- Restart Jellyfin once more and repeat one Live TV, one VOD, one series episode, and one STRM playback.
- Confirm there are no new plugin errors, leaked credentials, stuck network requests, orphaned restream sessions, or unexpected files.
- Run the complete Release test suite and release metadata validation on the exact commit to be tagged.
- Only after every item passes, tag that commit `v0.9.0` and run the publish workflow.
