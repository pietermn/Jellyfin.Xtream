# v0.9 release handoff

The v0.9 implementation is on `codex/v0.9.0`. Automated validation is green, but do not publish `v0.9.0` until the real Jellyfin 10.11 smoke test in [`V0.9_SMOKE_TEST.md`](V0.9_SMOKE_TEST.md) passes.

## Verified checkpoint — 2026-07-11

- Release build: zero warnings and errors.
- Test suite: 136 passed, zero failed/skipped.
- `git diff --check`, release-metadata validation for `v0.9.0`, and JavaScript syntax validation all pass.
- Public proxy URLs now capture the same configured URL as the grant fingerprint, eliminating an immediate-403 race during a concurrent configuration update.
- `*.home.arpa` private-provider origins are supported alongside IP literals, localhost, `.local`, and single-label LAN names; public hostnames still cannot resolve to private addresses.
- Legacy migration preview creation is serialized with quarantine, preventing concurrent previews for the same export kind from racing the one-time preview store.

## Log diagnosis to carry forward

- The latest Jellyfin.Xtream failures in the supplied log were caused by **both v0.8.11 and v0.8.12 being loaded side-by-side**. The evidence is the cross-assembly `PluginConfiguration` invalid cast plus duplicate `/Xtream/*` endpoints. The upgrade smoke test must preserve configuration while ensuring only the v0.9 assembly/plugin directory is loaded.
- The repeated `Jellyfin.Plugin.Streamyfin.PushNotifications.Events.ItemAddedService` null references are from the separate Streamyfin Push Notifications plugin, not Jellyfin.Xtream.
- The smoke checklist now contains an explicit v0.8.12 → v0.9 upgrade fixture: retain configuration, ensure only one plugin version is loaded, then save configuration and refresh the guide.

## Implemented

- Live TV remains a native Jellyfin tuner: it does not export STRM files. Cleanup rules and manual TV overrides now change the channel display names returned to Jellyfin.
- Cleanup rules support scoped, timeout-limited regex replacements for live channels/programs, VOD, series, seasons, episodes, categories, and filesystem paths.
- Live TV, catch-up, VOD, series, season, and episode requests validate provider IDs against the configured selection before fetching or opening a stream.
- Live TV media sources require Jellyfin's opening/closing callback, allowing the shared restream buffer to be used on Jellyfin 10.11.
- Client-visible URLs no longer contain Xtream credentials. Playback grants expire after 15 minutes; exported STRM files use an independently rotatable persistent resolver key.
- Anonymous proxy/resolver responses disable caching. Playback and persistent STRM keys can be rotated independently from the Credentials dashboard.
- Provider redirects are explicit, same-origin, and limited to five hops. Connections are DNS-approved and IP-pinned, unsafe metadata/non-global destinations are blocked, explicit LAN origins including `.home.arpa` remain supported, proxies are disabled, and credential-bearing request URIs are suppressed from .NET HTTP diagnostics.
- STRM reconciliation uses stable provider identities, atomic writes, and a manifest so cleanup touches only plugin-owned files.
- The VOD and Series dashboards provide preview-bound legacy v0.8 STRM quarantine. Reviewed files are hash-revalidated, moved without deletion, and recorded in an in-progress/final report. Old credential rotations and both historical v0.8 filename layouts are detected.
- Provider payload parsing tolerates malformed timestamps, null/absent collections, and ambiguous plain-text EPG titles.
- Provider cache loads coalesce safely, have bounded sizing and timeouts, and keep stalled response-body parsing cancellable.
- Dashboard settings remain saveable when category loading fails; TV overrides refresh safely.
- A plugin `Public Jellyfin URL` override is available for reverse-proxy deployments, is part of proxy-grant revocation state, and is captured consistently with playback/STRM grants.
- Manual release dispatch requires and validates an exact existing release tag.

- Add a Jellyfin integration-test fixture for the Live TV open/close callback path. Unit tests validate the contract flags, but the release gate remains the real-server smoke test.

Current hash, root, and reparse-point checks protect normal administrative use, but a process with concurrent filesystem write access is outside that guarantee.
