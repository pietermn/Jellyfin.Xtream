# v0.9 release handoff

The v0.9 implementation is on `codex/v0.9.0`. Do not publish `v0.9.0`: the final tail of audit fixes described below was deliberately stopped before compilation/testing, and the real Jellyfin 10.11 smoke test in [`V0.9_SMOKE_TEST.md`](V0.9_SMOKE_TEST.md) is still outstanding.

## Current stop point — 2026-07-11

The last verified combined tree passed a Release build with zero warnings and all 132 tests. After that run, the following uncommitted edits were made and **have not been compiled or tested**:

- Paginated the legacy migration preview so every candidate is reviewable instead of silently hiding paths after the first 200.
- Added Credentials-page save error handling so rejected public URLs do not leave a loading spinner active.
- Changed STRM normalization to apply both the content scope and `Filesystem` scope (`Vod|Filesystem`, `Series|Filesystem`, and `Episode|Filesystem`) and added a focused test.
- Allowed series providers to omit `category_id` while still rejecting an explicit category mismatch, with a focused authorization test.
- Made STRM export reject non-positive movie, series, and episode IDs and conservatively skip reconciliation after invalid provider data.
- Tightened legacy URL matching to require both username and password path segments.
- Marked oversized or reparse-point STRM paths as an incomplete migration scan and added an oversized-file test.
- Added migration-report outcomes for pending, moved, and changed-but-retained quarantine entries.

## Implemented

- Live TV remains a native Jellyfin tuner: it does not export STRM files. Cleanup rules and manual TV overrides now change the channel display names returned to Jellyfin.
- Cleanup rules support scoped, timeout-limited regex replacements for live channels/programs, VOD, series, seasons, episodes, categories, and filesystem paths.
- Live TV, catch-up, VOD, series, season, and episode requests validate provider IDs against the configured selection before fetching or opening a stream.
- Live TV media sources require Jellyfin's opening/closing callback, allowing the shared restream buffer to be used on Jellyfin 10.11.
- Client-visible URLs no longer contain Xtream credentials. Playback grants expire after 15 minutes; exported STRM files use an independently rotatable persistent resolver key.
- Anonymous proxy/resolver responses disable caching. Playback and persistent STRM keys can be rotated independently from the Credentials dashboard.
- Provider redirects are explicit, same-origin, and limited to five hops. Connections are DNS-approved and IP-pinned, unsafe metadata/non-global destinations are blocked, proxies are disabled, and credential-bearing request URIs are suppressed from .NET HTTP diagnostics.
- STRM reconciliation uses stable provider identities, atomic writes, and a manifest so cleanup touches only plugin-owned files.
- The VOD and Series dashboards provide preview-bound legacy v0.8 STRM quarantine. Reviewed files are hash-revalidated, moved without deletion, and recorded in an in-progress/final report. Old credential rotations and both historical v0.8 filename layouts are detected.
- Provider payload parsing tolerates malformed timestamps, null/absent collections, and ambiguous plain-text EPG titles.
- Provider cache loads coalesce safely, have bounded sizing and timeouts, and keep stalled response-body parsing cancellable.
- Dashboard settings remain saveable when category loading fails; TV overrides refresh safely.
- A plugin `Public Jellyfin URL` override is available for reverse-proxy deployments and is part of proxy-grant revocation state.
- Manual release dispatch requires and validates an exact existing release tag.
